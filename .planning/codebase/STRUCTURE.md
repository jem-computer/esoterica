# Codebase Structure

**Analysis Date:** 2026-01-21

## Directory Layout

```
esoterica/
├── .claude-plugin/          # Claude plugin manifest and metadata
│   └── marketplace.json     # Plugin registry configuration
└── .planning/               # Planning and documentation (git-ignored)
    └── codebase/           # Architecture analysis documents
```

## Directory Purposes

**.claude-plugin:**
- Purpose: Contains Claude plugin marketplace metadata and configuration
- Contains: Plugin manifest files in JSON format
- Key files: `marketplace.json` - the primary plugin registry

**.planning:**
- Purpose: Holds planning documents, architecture analysis, and development guides
- Contains: Markdown documentation files (.planning/codebase/)
- Key files: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md

**.planning/codebase:**
- Purpose: Repository for codebase analysis documents generated by GSD tools
- Contains: Markdown documentation analyzing code patterns, conventions, and architecture
- Key files: Generated analysis documents (STACK.md, INTEGRATIONS.md, ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md)

## Key File Locations

**Entry Points:**
- `/.claude-plugin/marketplace.json`: Primary configuration file that defines all plugins available in the marketplace

**Configuration:**
- `/.claude-plugin/marketplace.json`: Complete plugin registry with ownership, versioning, and source information

**Core Logic:**
- None (this is a configuration-only repository)

**Testing:**
- None (no test files in current structure)

## Naming Conventions

**Files:**
- Configuration files: lowercase with file extensions (marketplace.json)
- Documentation: UPPERCASE.md (ARCHITECTURE.md, STRUCTURE.md)

**Directories:**
- Hidden directories: dot-prefix with lowercase (`.claude-plugin`, `.planning`)
- Descriptive names: kebab-case or lowercase (codebase)

## Where to Add New Code

**New Plugin Registration:**
- Add entry to `plugins` array in `/.claude-plugin/marketplace.json`
- Include: name, source, description, version, author

**New Documentation:**
- Place in `/.planning/codebase/` directory
- Name: UPPERCASE.md (following GSD convention)

**Local Plugin Addition:**
- Create plugin directory structure parallel to `.claude-plugin`
- Update marketplace.json with local source reference: `"source": "./plugins/[plugin-name]"`

## Special Directories

**.claude-plugin:**
- Purpose: Standard Claude plugin metadata directory
- Generated: No (manually maintained)
- Committed: Yes (part of plugin distribution)

**.planning:**
- Purpose: Development planning and analysis documentation
- Generated: Yes (by GSD analysis tools)
- Committed: Typically yes (helps team alignment)
